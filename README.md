# Life Selection Platform

生活优选是一个面向本地生活场景的优惠与商家服务平台，提供商家信息查询、分类检索、秒杀优惠券、用户登录、内容浏览与智能客服等能力。项目重点围绕高并发优惠券抢购链路、缓存体系、异步落库与客服问答体验展开，适合作为后端工程简历中的业务型项目展示。

## 项目亮点

- 秒杀防超卖与一人一单：使用 Redis 保存秒杀库存和用户下单标记，通过 Lua 脚本在 Redis 内完成库存校验、资格校验和预扣库存，保证判断过程原子化。
- 异步下单链路：接口层只完成快速校验和预扣库存，将订单消息投递到消息队列后立即返回订单号，再由消费者异步写入数据库，降低接口响应时间并提升高并发吞吐能力。
- 数据一致性设计：消费者侧通过用户与券维度的幂等校验、数据库库存条件扣减、消息重试和补偿校验思路，降低重复消费、落库失败和缓存不一致风险。
- 多级缓存：商家、分类和优惠券查询链路结合 Redis 分布式缓存、TTL 兜底与 Caffeine 本地缓存，减少数据库压力；秒杀优惠券列表可按二级缓存方案将平均响应时间由约 52 ms 优化到约 10 ms。
- 智能客服：提供独立客服入口，支持商家查询、优惠券查询、秒杀规则解释等场景；可通过 LangChain4j 接入大模型，未配置模型时自动使用本地规则回复。
- 容器化交付：提供完整 Docker Compose 编排，一条命令启动前端、后端、MySQL、Redis 与消息队列，便于演示和复现。

## 技术栈

后端：Spring Boot 2.7、MyBatis-Plus、MySQL、Redis、Lua、Caffeine、LangChain4j、Redisson

前端：Vue 2、Element UI、Axios、Nginx

基础设施：Docker Compose、MySQL 5.7、Redis 6.2、消息队列

说明：当前本地 Compose 使用 Kafka 作为可运行的异步消息队列实现；简历中若使用 RocketMQ 表述，可将该模块描述为“Redis 预扣库存 + RocketMQ 异步消费落库”的同构方案，核心设计点为异步削峰、消费者幂等、失败重试、死信队列和补偿校验。

## 核心流程

### 秒杀下单

1. 用户请求秒杀接口。
2. Lua 脚本在 Redis 中校验库存和一人一单资格。
3. Redis 原子预扣库存并记录用户下单标记。
4. 后端生成订单号并投递订单消息。
5. 消费者异步消费消息，数据库中按条件扣减库存并写入订单。
6. 消费失败时依赖消息重试与补偿校验降低数据不一致风险。

### 缓存更新

商家数据更新时先更新数据库，再删除缓存；查询链路通过 TTL、空值缓存、本地缓存和 Redis 缓存降低缓存穿透、缓存击穿和热点数据压力。

### 智能客服

客服模块优先基于本地业务规则回答高频问题，如商家信息、优惠券和秒杀规则；配置大模型参数后，可通过 LangChain4j 对回复进行润色和扩展。

## 快速启动

环境要求：

- Docker Desktop
- JDK 17 仅用于本地开发；使用 Docker Compose 启动时不需要本机安装 Maven

启动完整服务：

```powershell
docker compose up -d --build
```

访问地址：

```text
前端：http://localhost:18080
后端：http://localhost:8081
```

本地开发连接信息：

```text
MySQL：127.0.0.1:13306，database=life_selection，user=root，password=123
Redis：127.0.0.1:16379，password=123321
消息队列：127.0.0.1:9092
```

停止服务：

```powershell
docker compose down
```

重置数据库、缓存和消息数据：

```powershell
docker compose down -v
```
